# Meeting Intelligence Dashboard - MVP開発計画

## 📋 プロジェクト概要

### コンセプト
**「会議のデジタルトランスフォーメーション（DX）を実現する、プロジェクトの心拍数モニター」**

### 背景
Bunのエンジニアが1週間で100万行を書き換える裏には、「誰かの確認を待つ」「空気を読んで黙る」時間がゼロという働き方がある。日本的な丁寧さを維持しながらAI的なスピードを手に入れるため、会議の構造改革が必要。

### 解決する課題
- ❌ 会議が長すぎる（準備不足で説明に時間を取られる）
- ❌ 決定事項が曖昧（誰が何をいつまでにやるか不明確）
- ❌ 特定の人だけが話す（多様な意見が出ない）
- ❌ 沈黙時間が多い（心理的安全性の欠如）
- ❌ 会議の効果測定ができない（改善のPDCAが回らない）

### 提供価値
- ⏱️ **会議時間を50%削減**（事前準備の促進）
- 📊 **決定密度の可視化**（生産性の定量評価）
- 🎯 **非同期コミュニケーションへのシフト**（Markdown活用）
- 📈 **継続的な改善サイクル**（データドリブンな会議運営）

## 🎯 MVP（最小実行可能製品）の範囲

### フェーズ1: 簡易版（今回実装）
**目的**: 最速で価値検証を行う

#### 実装機能
1. **議事録テキスト入力**
   - 手動で会議の議事録をテキストエリアに貼り付け
   - 事前アジェンダも別途入力可能

2. **AI分析エンジン**
   - OpenAI API（GPT-4）による多角的分析
   - 5つのKPI自動採点
   - 具体的な改善提案の生成

3. **評価レポート生成**
   - Markdown形式での構造化レポート
   - ダウンロード機能（.md形式）

4. **ダッシュボード表示**
   - KPIスコアの可視化（レーダーチャート）
   - 時系列トレンド表示（複数回の会議を比較）

#### 技術スタック
- **フロントエンド**: HTML + CSS + JavaScript（Vanilla JS）
- **AI**: OpenAI API（GPT-4）
- **可視化**: Chart.js
- **デプロイ**: 単一HTMLファイル（ローカル実行可能）

#### 制約事項
- 音声認識は含まない（将来フェーズで実装）
- Box連携は含まない（将来フェーズで実装）
- リアルタイム分析は含まない（バッチ処理のみ）

## 📊 KPI評価項目の定義

### 1. 事前準備度（Pre-Meeting Readiness）
**定義**: 会議前にどれだけ準備されていたか

**評価基準**:
- アジェンダの明確さ（5点満点）
- 事前資料の充実度（5点満点）
- 参加者への事前共有状況（5点満点）

**計算式**:
```
事前準備度スコア = (アジェンダ評価 + 資料評価 + 共有評価) / 15 × 100
```

**改善アクション例**:
- 「アジェンダが抽象的です。具体的な議論ポイントを3つ明記しましょう」
- 「事前資料がありません。Box上に資料を配置し、48時間前に共有しましょう」

---

### 2. 決定密度（Decision Density）
**定義**: 会議時間に対する決定事項の数

**評価基準**:
- 明確な決定事項の数
- 各決定の具体性（担当者・期限の明記）
- 会議時間との比率

**計算式**:
```
決定密度スコア = (決定事項数 × 具体性係数) / 会議時間(分) × 10
```

**改善アクション例**:
- 「60分の会議で決定事項が1件のみ。目標は3件以上です」
- 「決定事項に担当者が明記されていません。次回は必ず指名しましょう」

---

### 3. 発言の多様性（Voice Diversity）
**定義**: 参加者全員が発言しているか、多様な意見が出ているか

**評価基準**:
- 発言者の分散度（ジニ係数）
- 反対意見・批判的意見の有無
- 沈黙している参加者の割合

**計算式**:
```
多様性スコア = (1 - ジニ係数) × 反対意見係数 × 100
```

**改善アクション例**:
- 「Aさんが全体の70%を話しています。他の参加者にも意見を求めましょう」
- 「全員が賛成しています。リスクを指摘する役割を設定しましょう」

---

### 4. サイレントコスト（Silent Cost）
**定義**: 誰も発言していない沈黙時間の合計

**評価基準**:
- 沈黙時間の合計（秒）
- 会議時間に対する割合
- 沈黙の文脈（考える時間 vs 気まずい沈黙）

**計算式**:
```
サイレントコストスコア = 100 - (沈黙時間 / 会議時間 × 100)
```

**改善アクション例**:
- 「15分間の沈黙がありました。ファシリテーターが質問で場を動かしましょう」
- 「沈黙が多いのは心理的安全性の問題かもしれません」

---

### 5. アジェンダ遵守度（Agenda Adherence）
**定義**: 予定していた議題から脱線せずに進行できたか

**評価基準**:
- 各アジェンダ項目の消化率
- 予定外の議論に費やした時間
- 時間配分の適切性

**計算式**:
```
遵守度スコア = (消化したアジェンダ数 / 予定アジェンダ数) × (1 - 脱線時間率) × 100
```

**改善アクション例**:
- 「アジェンダ3が未消化です。次回に持ち越すか、別途時間を設定しましょう」
- 「予定外の議論に20分使いました。緊急度が高ければ別会議を設定しましょう」

### AI分析プロンプト設計

#### システムプロンプト
```
あなたは会議分析の専門家です。
提供された議事録とアジェンダを分析し、以下の5つのKPIを評価してください：

1. 事前準備度（0-100点）
2. 決定密度（0-100点）
3. 発言の多様性（0-100点）
4. サイレントコスト削減（0-100点）
5. アジェンダ遵守度（0-100点）

各KPIについて、スコアの根拠と具体的な改善提案を提供してください。
```

#### ユーザープロンプト（テンプレート）
```
# 会議情報
- 会議名: {meeting_title}
- 日時: {meeting_date} {meeting_time}
- 参加者: {participants}
- 時間: {duration_minutes}分

# 事前アジェンダ
{agenda_text}

# 議事録
{transcript_text}

# 分析指示
上記の会議について、5つのKPIを評価し、JSON形式で結果を返してください。
```
