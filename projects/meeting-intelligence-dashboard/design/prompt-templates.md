# AI分析プロンプト設計 - Meeting Intelligence Dashboard

## 📋 概要

本ドキュメントでは、OpenAI GPT-4を使用した会議分析のためのプロンプトテンプレートを定義します。

---

## 🎯 プロンプト設計の原則

### 1. 構造化された出力
- JSON形式で構造化されたデータを返す
- スコアは必ず0-100の範囲
- 根拠を明確に記述

### 2. 日本語最適化
- 日本語の会議特有の表現を理解
- 敬語・謙譲語の使用を考慮
- 「えーと」「あの」などのフィラーを適切に処理

### 3. 具体的な改善提案
- 抽象的な提案ではなく、具体的なアクション
- 期待効果を数値で示す
- 優先順位を明確に

---

## 🤖 システムプロンプト

```
あなたは会議分析の専門家です。日本企業の会議文化を深く理解し、以下の5つのKPIを評価します：

ただし、MVPではテキスト議事録からの推定に限界があるため、推定値と明示的に確認できる事実を区別して出力してください。特に発言割合、沈黙時間、アジェンダごとの実時間は推定値として扱ってください。

1. 事前準備度（Pre-Meeting Readiness）: 0-100点
2. 決定密度（Decision Density）: 0-100点
3. 発言の多様性（Voice Diversity）: 0-100点
4. サイレントコスト削減（Silent Cost Reduction）: 0-100点
5. アジェンダ遵守度（Agenda Adherence）: 0-100点

## 評価の観点

### 事前準備度
- アジェンダの明確さ（目的・期待する成果が明記されているか）
- 事前資料の充実度（データ・図表が含まれているか）
- 参加者への事前共有状況（事前質問があるか）

### 決定密度
- 決定事項の数（会議時間に対して適切か）
- 決定の具体性（What/Who/When/Whyが明確か）
- 実行可能性（担当者と期限が明記されているか）

### 発言の多様性
- 発言の分散度（特定の人だけが話していないか）
- 反対意見・批判的意見の有無（建設的な議論があるか）
- 沈黙参加者の割合（全員が発言しているか）

### サイレントコスト削減
- 沈黙時間の割合（会議時間に対する比率）
- 沈黙の文脈（考える時間 vs 気まずい沈黙）
- ファシリテーションの質（沈黙を適切に扱えているか）

### アジェンダ遵守度
- アジェンダ消化率（予定した議題を完了できたか）
- 脱線時間の割合（予定外の議論に費やした時間）
- 時間配分の適切性（各議題の予定時間を守れたか）

## 出力形式

必ずJSON形式で以下の構造で出力してください：

```json
{
  "overall_score": 0-100の数値,
  "grade": "S" | "A" | "B" | "C" | "D" | "E",
  "kpi_scores": {
    "pre_meeting_readiness": { "score": 0-100, "breakdown": {...}, "reasoning": "根拠" },
    "decision_density": { "score": 0-100, "breakdown": {...}, "reasoning": "根拠" },
    "voice_diversity": { "score": 0-100, "breakdown": {...}, "reasoning": "根拠" },
    "silent_cost_reduction": { "score": 0-100, "breakdown": {...}, "reasoning": "根拠" },
    "agenda_adherence": { "score": 0-100, "breakdown": {...}, "reasoning": "根拠" }
  },
  "analysis_confidence": "high" | "medium" | "low",
  "data_quality_flags": [...],
  "assumptions": [...],
  "decisions": [...],
  "voice_analysis": {...},
  "silent_periods": [...],
  "agenda_coverage": {...},
  "recommendations": [...],
  "highlights": {
    "strengths": [...],
    "weaknesses": [...],
    "risks": [...],
    "opportunities": [...]
  }
}
```

## 重要な注意事項

1. **客観性**: 感情的な評価ではなく、事実に基づいた分析
2. **建設的**: 批判だけでなく、具体的な改善策を提示
3. **文化的配慮**: 日本の会議文化（コンセンサス重視、階層意識）を理解
4. **実行可能性**: 提案は現実的で、すぐに実行できるもの
5. **数値化**: 可能な限り定量的な評価と期待効果を示す
6. **信頼度の明示**: 入力品質が低い場合は `analysis_confidence` を下げ、`data_quality_flags` と `assumptions` に理由を記載する
7. **形式依存性の考慮**: `名前: 発言` 形式でない議事録では、話者分析の精度が下がることを明示する
```

```
以下の会議について、5つのKPIを評価し、JSON形式で結果を返してください。

# 会議情報
- 会議名: {{meeting_title}}
- 日時: {{meeting_date}} {{meeting_time}}
- 参加者: {{participants}}
- 会議時間: {{duration_minutes}}分

# 事前アジェンダ
{{agenda_text}}

# 議事録
{{transcript_text}}

# 分析指示
上記の会議について、以下を分析してください。

なお、議事録形式に応じて分析精度が変わります。
- `名前: 発言` 形式なら話者分析を詳細に行う
- 自由記述の場合は、話者分析・沈黙分析・時間配分分析の信頼度を下げる

1. 各KPIのスコア（0-100点）と根拠
2. 分析全体の信頼度（high/medium/low）
3. データ品質上の注意点（data_quality_flags）
4. 推定に使った前提（assumptions）
5. 決定事項の一覧（What/Who/When/Whyを明記）
6. 発言分析（各参加者の発言割合、反対意見の有無）
7. 沈黙時間の分析（文脈を含む、推定値なら明示）
8. アジェンダの消化状況
9. 具体的な改善提案（優先度付き）

必ずJSON形式で、構造化されたデータとして出力してください。
```

### テンプレート2: 詳細分析（アジェンダあり）

```
以下の会議について、事前アジェンダと実際の議事録を比較し、詳細な分析を行ってください。

# 会議情報
- 会議名: {{meeting_title}}
- 日時: {{meeting_date}} {{meeting_time}}
- 参加者: {{participants_list}}
- 会議時間: {{duration_minutes}}分
- ファシリテーター: {{facilitator}}

# 事前アジェンダ（予定）
{{#each agenda_items}}
{{order}}. {{title}} ({{allocated_minutes}}分)
   目的: {{description}}
   担当: {{owner}}
{{/each}}

# 事前資料
{{#if materials_provided}}
以下の資料が{{advance_notice_hours}}時間前に共有されました：
{{#each materials}}
- {{title}}: {{url}}
{{/each}}
{{else}}
事前資料は共有されていません。
{{/if}}

# 事前質問
{{#if pre_questions}}
以下の事前質問がありました：
{{#each pre_questions}}
- {{author}}: {{question}}
{{/each}}
{{else}}
事前質問はありませんでした。
{{/if}}

# 議事録（実際の会議内容）
{{transcript_text}}

# 分析指示

## 1. 事前準備度の評価
- アジェンダの明確さ（目的・期待する成果が明記されているか）
- 事前資料の充実度（{{advance_notice_hours}}時間前の共有は適切か）
- 事前質問の有無（参加者の準備状況）

## 2. 決定密度の評価
- 決定事項を抽出し、以下を評価：
  * What（何を）: 具体的か
  * Who（誰が）: 担当者が明記されているか
  * When（いつまで）: 期限が設定されているか
  * Why（なぜ）: 理由・背景が記録されているか
- 各決定の具体性スコア（0-10点）

## 3. 発言の多様性の評価
- 各参加者の発言割合を推定
- ジニ係数を計算（0=完全に均等、1=一人だけが話す）
- 反対意見・批判的意見の有無
- 沈黙している参加者の特定

## 4. サイレントコストの評価
- 沈黙時間を特定（「（沈黙）」「...」などの表現から）
- 各沈黙の文脈を分析：
  * 良い沈黙: 考える時間、資料確認
  * 悪い沈黙: 質問後に誰も答えない、気まずい雰囲気
- 総沈黙時間と会議時間の比率

## 5. アジェンダ遵守度の評価
- 各アジェンダ項目の消化状況：
  * 完了/未完了
  * 予定時間 vs 実際の時間
  * 時間超過の理由
- 予定外の議論（脱線）の特定と時間
- 時間配分の適切性

## 6. 改善提案
各KPIについて、以下の形式で提案：
- 優先度（high/medium/low）
- 現状の問題点
- 具体的なアクション（3つ以内）
- 期待効果（時間削減、スコア向上など）

必ずJSON形式で、データモデルに従った構造化されたデータとして出力してください。
```

### テンプレート3: 簡易分析（アジェンダなし）

```
以下の会議について、議事録のみから分析を行ってください。

# 会議情報
- 会議名: {{meeting_title}}
- 日時: {{meeting_date}} {{meeting_time}}
- 参加者: {{participants}}
- 会議時間: {{duration_minutes}}分

# 議事録
{{transcript_text}}

# 分析指示

事前アジェンダが提供されていないため、以下の点に注意して分析してください：
また、このケースでは `data_quality_flags` に `agenda_missing` を含め、必要に応じて `analysis_confidence` を下げてください。

1. **事前準備度**: アジェンダがないこと自体が減点要因。議事録から「準備不足」の兆候を探す
   - 「背景から説明すると」「まず前提として」などの表現
   - 資料の言及がない
   - 参加者が内容を理解していない様子

2. **決定密度**: 議事録から決定事項を抽出
   - 「決定しました」「承認します」「〜することにします」などのキーワード
   - 各決定の具体性を評価

3. **発言の多様性**: 発言者を特定し、割合を推定
   - 「〇〇:」の形式から発言者を抽出
   - 文字数から発言時間を推定

4. **サイレントコスト**: 沈黙の記述を探す
   - 「（沈黙）」「...」「しばらく間があり」などの表現
   - 数値化する場合は推定値であることを前提にする

5. **アジェンダ遵守度**: アジェンダがないため、議論の一貫性を評価
   - 話題の転換が多いか
   - 脱線が多いか

6. **信頼度評価**
   - 分析結果全体の信頼度
   - 品質上の制約
   - 推定時の仮定

必ずJSON形式で出力してください。
```

## 🔍 分析ロジックの詳細

### 1. 発言者の特定と発言時間の推定

```javascript
// プロンプトに含める分析ロジックの説明

/**
 * 発言者の特定方法：
 * - 「田中:」「佐藤:」などのパターンで発言者を抽出
 * - 各発言者の文字数をカウント
 * - 文字数から発言時間を推定（日本語: 約300文字/分）
 * 
 * ジニ係数の計算：
 * 1. 各発言者の発言割合を計算
 * 2. 割合を昇順にソート
 * 3. ジニ係数 = 1 - Σ(2 * (n - i + 0.5) * p_i) / n
 *    where: n = 参加者数, p_i = i番目の発言割合
 */
```

### 2. 決定事項の抽出

```javascript
// プロンプトに含める決定事項の判定基準

/**
 * 決定を示すキーワード：
 * - 「決定しました」「承認します」「合意しました」
 * - 「〜することにします」「〜で進めます」
 * - 「〜を採用します」「〜に決まりました」
 * 
 * 具体性の評価（各項目0-10点）：
 * - What（何を）: 行動が具体的か
 * - Who（誰が）: 担当者が明記されているか
 * - When（いつまで）: 期限が設定されているか（「来週」は5点、「1月31日」は10点）
 * - Why（なぜ）: 理由・背景が記録されているか
 * 
 * 曖昧な表現（減点対象）：
 * - 「検討します」「考えておきます」「後で決めます」
 * - 「誰かやっておいて」「いい感じで」
 */
```

### 3. 沈黙時間の推定

```javascript
// プロンプトに含める沈黙の判定基準

/**
 * 沈黙を示す表現：
 * - 「（沈黙）」「...」「しばらく間があり」
 * - 「誰も答えない」「沈黙が続く」
 * 
 * 沈黙の文脈分析：
 * 
 * 良い沈黙（減点なし）：
 * - 直前に「少し考えさせてください」
 * - 直前に「この図を見てください」
 * - 直前に「質問はありますか？」（意図的な間）
 * 
 * 悪い沈黙（減点対象）：
 * - 質問後に誰も答えない
 * - 「どうでしょうか？」の後の沈黙
 * - 気まずい雰囲気の記述
 * 
 * 沈黙時間の推定：
 * - 「（沈黙）」のみ: 30秒と推定
 * - 「しばらく沈黙」: 2分と推定
 * - 「長い沈黙」: 5分と推定
 */
```

### 4. アジェンダの消化状況

```javascript
// プロンプトに含めるアジェンダ分析の基準

/**
 * アジェンダ項目の完了判定：
 * 
 * 完了（completed）：
 * - 議事録に該当する議論が含まれている
 * - 決定事項または結論が記録されている
 * 
 * 未完了（incomplete）：
 * - 議事録に言及がない
 * - 「時間切れ」「次回に持ち越し」の記述
 * 
 * 時間配分の分析：
 * - 各アジェンダの議論開始・終了時刻を推定
 * - 予定時間との差分を計算
 * - 超過率 = (実際の時間 - 予定時間) / 予定時間 × 100
 * 
 * 脱線の判定：
 * - アジェンダに含まれない話題
 * - 「話は変わりますが」「ついでに」「そういえば」などのキーワード
 */
```

## 📊 スコアリングアルゴリズム

### 総合スコアの計算

```
総合スコア = (事前準備度 × 0.25) + (決定密度 × 0.25) + 
             (発言多様性 × 0.15) + (サイレント削減 × 0.15) + 
             (アジェンダ遵守 × 0.20)
```

### グレードの判定

```
S: 90-100点 - 理想的な会議
A: 80-89点  - 優秀な会議
B: 70-79点  - 良好な会議
C: 60-69点  - 改善の余地あり
D: 50-59点  - 要改善
E: 0-49点   - 抜本的な見直しが必要
```

## 🎯 改善提案の生成ロジック

### 優先度の決定

```javascript
// プロンプトに含める優先度判定の基準

/**
 * 優先度: high（緊急改善が必要）
 * - スコアが40点未満
 * - 目標値との差が-20点以上
 * - 会議の生産性に直接影響
 * 
 * 優先度: medium（改善推奨）
 * - スコアが40-70点
 * - 目標値との差が-10〜-19点
 * - 改善により大きな効果が期待できる
 * 
 * 優先度: low（さらなる改善）
 * - スコアが70点以上
 * - 目標値との差が-10点未満
 * - 既に良好だが、さらに向上可能
 */
```

### 期待効果の算出

```javascript
// プロンプトに含める効果推定の基準

/**
 * 時間削減の推定：
 * 
 * 事前準備の改善:
 * - 資料を事前共有 → 説明時間を50%削減
 * - 60分会議で説明が20分 → 10分削減
 * 
 * 決定プロセスの改善:
 * - 決定テンプレート使用 → 決定時間を30%短縮
 * - 曖昧な議論が15分 → 5分削減
 * 
 * アジェンダ遵守の改善:
 * - タイムキーパー設定 → 脱線時間を70%削減
 * - 脱線が15分 → 10分削減
 * 
 * スコア向上の推定：
 * - 各改善アクションごとに+5〜20点の向上を見込む
 * - 複数の改善を組み合わせることで相乗効果
 */
```

## 🧪 プロンプトのテスト

### テストケース1: 理想的な会議

```
入力:
- 事前アジェンダ: 明確（目的・期待する成果あり）
- 事前資料: 48時間前に共有
- 議事録: 全員が発言、決定事項3件（具体的）、沈黙なし
- アジェンダ: 全て消化、時間通り

期待される出力:
- 総合スコア: 85-95点
- グレード: A または S
- 改善提案: 優先度 low のみ
```

### テストケース2: 問題のある会議

```
入力:
- 事前アジェンダ: なし
- 事前資料: なし
- 議事録: 一人だけが話す、決定事項0件、沈黙多数
- アジェンダ: 脱線が多い

期待される出力:
- 総合スコア: 30-45点
- グレード: E または D
- 改善提案: 優先度 high が複数
```

### テストケース3: 平均的な会議

```
入力:
- 事前アジェンダ: あるが抽象的
- 事前資料: 前日に共有
- 議事録: 発言に偏りあり、決定事項2件（やや曖昧）、沈黙少々
- アジェンダ: 一部未消化

期待される出力:
- 総合スコア: 60-70点
- グレード: C または B
- 改善提案: 優先度 medium が中心
```

 *   "analysis_confidence": "low",
 *   "data_quality_flags": ["speakers_not_clearly_identified"]

### 3. レスポンスの検証

```javascript
// 返されたJSONの検証
function validateResponse(response) {
  const required = [
    'overall_score',
    'grade',
    'kpi_scores',
    'recommendations'
  ];
  
  for (const field of required) {
    if (!(field in response)) {
      throw new Error(`必須フィールド ${field} がありません`);
    }
  }
  
  // スコアの範囲チェック
  if (response.overall_score < 0 || response.overall_score > 100) {
    throw new Error('スコアは0-100の範囲である必要があります');
  }
  
  return true;
}
```
