# KPI定義書 - Meeting Intelligence Dashboard

## 📊 概要

本ドキュメントでは、Meeting Intelligence Dashboardで使用する5つのKPI（重要業績評価指標）について、評価基準、計算方法、改善アクションを詳細に定義します。

---

| # | KPI名 | 英語名 | 目標値 | 重要度 |
|---|-------|--------|--------|--------|
| 1 | 事前準備度 | Pre-Meeting Readiness | 80点以上 | ⭐⭐⭐ |
| 2 | 決定密度 | Decision Density | 70点以上 | ⭐⭐⭐ |
| 3 | 発言の多様性 | Voice Diversity | 75点以上 | ⭐⭐ |

## 1️⃣ 事前準備度（Pre-Meeting Readiness）

### 定義
会議開始前にどれだけ準備が整っていたかを評価する指標。事前準備が充実していれば、会議時間を説明ではなく議論に使える。

### なぜ重要か
- 会議時間の50%は「説明」に費やされている（調査データ）
- 事前に資料を読んでおけば、会議は「決定」に集中できる
- Bunのような高速開発チームは、非同期での情報共有が徹底されている

### 評価項目

#### 1.1 アジェンダの明確さ（0-35点）
**評価基準**:
- **30-35点**: 各議題に「目的」「期待する成果」「時間配分」が明記
- **20-29点**: 議題は列挙されているが、詳細が不足
- **10-19点**: 議題が抽象的（例: "進捗確認"のみ）
- **0-9点**: アジェンダが存在しない、または直前に共有

**良い例**:
```markdown
## アジェンダ
1. プロジェクト進捗報告（10分）
   - 目的: 各タスクの完了状況を共有
   - 期待する成果: 遅延タスクの特定と対策決定
   
2. 技術的課題の検討（30分）
   - 目的: API設計の方針を決定
   - 期待する成果: 設計案の承認と次週の実装開始
```

**悪い例**:
```markdown
## アジェンダ
1. 進捗確認
2. 課題について
3. その他
```

#### 1.2 事前資料の充実度（0-35点）
**評価基準**:
- **30-35点**: 必要な資料が全て揃い、48時間前に共有済み
- **20-29点**: 資料はあるが、24時間前の共有
- **10-19点**: 資料が不完全、または当日共有
- **0-9点**: 資料が存在しない

**チェックリスト**:
- [ ] データ・数値が含まれている
- [ ] 図表・ビジュアルで理解しやすい
- [ ] 事前に質問・コメントができる形式（Box、Notion等）
- [ ] 48時間前に共有されている

#### 1.3 参加者への事前共有状況（0-30点）
**評価基準**:
- **25-30点**: 全参加者が資料を確認済み、事前質問あり
- **15-24点**: 過半数が確認済み
- **5-14点**: 一部のみ確認
- **0-4点**: 誰も確認していない

**測定方法**:
- Box等のアクセスログ
- 事前質問・コメントの有無
- 会議冒頭での「資料は見ましたか？」の回答

### 計算式
```
事前準備度スコア = アジェンダ評価(35) + 資料評価(35) + 共有評価(30)
```

### AI分析のポイント
```javascript
// 議事録から以下を検出
const indicators = {
  poor_preparation: [
    "資料を見ていない",
    "これから説明します",
    "背景から説明すると",
    "まず前提として"
  ],
  good_preparation: [
    "事前に共有した資料の通り",
    "質問いただいた点について",
    "資料を見ていただいた前提で"
  ]
};
```

### 改善アクション

#### スコア 0-40点（要改善）
```
🚨 緊急改善が必要です

【即座に実施】
1. 会議の48時間前までにアジェンダと資料をBoxに配置
2. アジェンダに「目的」「期待する成果」を明記
3. 参加者に事前確認を依頼（Teamsメンション）

【期待効果】
- 会議時間を30分短縮
- 説明時間を80%削減
```

#### スコア 41-70点（改善の余地あり）
```
⚠️ もう一歩で理想的な準備です

【推奨アクション】
1. 資料に「事前に考えてほしいポイント」を追記
2. 共有時に「ここだけは読んでください」を明示
3. 事前質問を促す（質問がない=理解不足の可能性）

【期待効果】
- 会議の質が向上
- 決定までのスピードが2倍に
```

#### スコア 71-100点（優秀）
```
✅ 素晴らしい準備です！

【さらなる改善】
1. 事前質問への回答も資料に追記（他の人も見れる）
2. 過去の会議資料とのリンクを追加
3. この準備プロセスをテンプレート化

【期待効果】
- チーム全体の標準が向上
- 新メンバーも同じ品質を実現
```

## 2️⃣ 決定密度（Decision Density）

### 定義
会議時間に対する「明確な決定事項」の数と質を評価する指標。会議の最大の目的は「決める」こと。

### なぜ重要か
- 決定がない会議は「情報共有」で終わる（メールで済む）
- 決定が曖昧だと、後で「あれは決まったのか？」となる
- 1週間で100万行を書き換えるには、決定のスピードが命

### 評価項目

#### 2.1 決定事項の数（基礎点）
**評価基準**:
- **60分会議の目標**: 3-5件の決定
- **30分会議の目標**: 2-3件の決定

**計算式**:
```
基礎点 = min(決定事項数 / 目標数 × 50, 50)
```

#### 2.2 決定の具体性（0-50点）
各決定事項について以下を評価:

| 要素 | 配点 | 評価基準 |
|------|------|----------|
| **What（何を）** | 15点 | 具体的な行動が明確か |
| **Who（誰が）** | 15点 | 担当者が明記されているか |
| **When（いつまで）** | 15点 | 期限が設定されているか |
| **Why（なぜ）** | 5点 | 理由・背景が記録されているか |

**良い決定の例**:
```markdown
## 決定事項
1. 新機能「ダッシュボード」の開発を開始する
   - 担当: 田中さん（フロントエンド）、佐藤さん（バックエンド）
   - 期限: 2024-01-31（MVP完成）
   - 理由: ユーザーからの要望が多く、競合優位性につながる
   - 次回確認: 2024-01-22の定例会議
```

**悪い決定の例**:
```markdown
## 決定事項
1. ダッシュボードを作る
   - 誰かやっておいて
   - なるべく早く
```

### 計算式
```
決定密度スコア = (基礎点 + 具体性平均点) × 時間係数

時間係数 = {
  60分以内: 1.0
  61-90分: 0.9
  91-120分: 0.8
  121分以上: 0.7
}
```

### AI分析のポイント
```javascript
// 決定を示すキーワード
const decision_keywords = [
  "決定しました",
  "承認します",
  "〜することにします",
  "〜で進めます",
  "合意しました"
];

// 曖昧な表現（減点対象）
const vague_expressions = [
  "検討します",
  "考えておきます",
  "後で決めます",
  "誰かやっておいて",
  "いい感じで"
];
```

### 改善アクション

#### スコア 0-40点（要改善）
```
🚨 会議が「決定」につながっていません

【問題の可能性】
- 議論が発散している
- 決定権者が不在
- 情報不足で判断できない

【即座に実施】
1. 会議の最後5分を「決定事項の確認」に充てる
2. 決定テンプレートを使う（What/Who/When）
3. 決定できない場合は「次回までに〇〇を調査」と明記

【期待効果】
- 会議の生産性が3倍に
- 手戻りが80%削減
```

#### スコア 41-70点（改善の余地あり）
```
⚠️ 決定はあるが、具体性が不足

【推奨アクション】
1. 決定事項に必ず担当者を指名（その場で）
2. 期限は「〇月〇日」と日付で（「来週」は NG）
3. 決定の背景も記録（後で「なぜ？」となる）

【期待効果】
- 実行率が90%以上に
- 次回会議での確認がスムーズ
```

#### スコア 71-100点（優秀）
```
✅ 決定のスピードと質が高いです！

【さらなる改善】
1. 決定事項を即座にタスク管理ツールに登録
2. 決定の影響範囲を明記（他チームへの連携）
3. 決定の成功基準を設定（どうなれば成功か）

【期待効果】
- プロジェクト全体の速度が向上
- 決定の質がさらに向上
```

## 3️⃣ 発言の多様性（Voice Diversity）

### 定義
参加者全員が発言し、多様な意見（特に反対意見）が出ているかを評価する指標。

### なぜ重要か
- 一人だけが話す会議は「報告会」（会議の必要なし）
- 反対意見がない=リスクの見落とし
- 心理的安全性の指標でもある

### 評価項目

#### 3.1 発言の分散度（0-50点）
**ジニ係数**を使用して発言の偏りを測定:

```
ジニ係数 = 0.0（完全に均等） ～ 1.0（一人だけが話す）

発言分散スコア = (1 - ジニ係数) × 50
```

**例**:
```
参加者3名、60分会議の場合

【理想的】
- Aさん: 20分（33%）
- Bさん: 20分（33%）
- Cさん: 20分（33%）
→ ジニ係数 ≈ 0.0、スコア = 50点

【偏りあり】
- Aさん: 45分（75%）
- Bさん: 10分（17%）
- Cさん: 5分（8%）
→ ジニ係数 ≈ 0.5、スコア = 25点
```

#### 3.2 反対意見・批判的意見（0-30点）
**評価基準**:
- **25-30点**: 建設的な反対意見が複数あり、議論が深まった
- **15-24点**: 質問はあるが、反対意見は少ない
- **5-14点**: 全員が賛成（リスク）
- **0-4点**: 誰も意見を言わない

**検出キーワード**:
```javascript
const critical_opinions = [
  "懸念があります",
  "リスクとして",
  "別の視点では",
  "反対意見ですが",
  "デメリットは",
  "〜の場合はどうなりますか"
];
```

#### 3.3 沈黙参加者の割合（0-20点）
**評価基準**:
- **18-20点**: 全員が発言（沈黙者0%）
- **12-17点**: 80%以上が発言
- **6-11点**: 50-79%が発言
- **0-5点**: 半数以上が沈黙

### 計算式
```
多様性スコア = 発言分散(50) + 反対意見(30) + 発言率(20)
```

### AI分析のポイント
```javascript
// 発言者の特定
const speaker_pattern = /^([^:：]+)[:：]/gm;

// 発言時間の推定（文字数から）
const estimate_duration = (text) => {
  const chars = text.length;
  const words_per_minute = 300; // 日本語の平均
  return chars / words_per_minute;
};

// 反対意見の検出
const detect_critical_opinion = (text) => {
  return critical_opinions.some(keyword => 
    text.includes(keyword)
  );
};
```

### 改善アクション

#### スコア 0-40点（要改善）
```
🚨 特定の人だけが話しています

【問題の可能性】
- 心理的安全性が低い
- 一部の人が会議を支配している
- 参加者が準備不足で発言できない

【即座に実施】
1. ファシリテーターが意図的に全員に振る
   「〇〇さん、どう思いますか？」
2. 「反対意見を歓迎する」と明言
3. 事前に意見を集める（匿名でも可）

【期待効果】
- リスクの早期発見
- チーム全体の当事者意識向上
```

#### スコア 41-70点（改善の余地あり）
```
⚠️ 発言はあるが、批判的意見が少ない

【推奨アクション】
1. 「デビルズ・アドボケイト」役を設定
   （意図的に反対意見を出す役割）
2. 「このプランの弱点は？」と問いかける
3. 沈黙している人に「質問はありますか？」

【期待効果】
- 意思決定の質が向上
- 後での「やっぱり問題だった」が減る
```

#### スコア 71-100点（優秀）
```
✅ 健全な議論ができています！

【さらなる改善】
1. 反対意見を出した人を称賛する文化
2. 「建設的な批判」のガイドライン作成
3. 新メンバーも発言しやすい雰囲気づくり

【期待効果】
- イノベーションが生まれやすい
- チームの心理的安全性が向上
```

## 4️⃣ サイレントコスト削減（Silent Cost Reduction）

### 定義
誰も発言していない「沈黙時間」の合計を測定し、削減を目指す指標。

### なぜ重要か
- 沈黙時間 = 全員の時間が無駄になっている
- 60分会議で10分沈黙 = 参加者5名なら50分の損失
- 沈黙の理由を分析することで、会議の問題が見える

### 評価項目

#### 4.1 沈黙時間の割合（0-60点）
**評価基準**:
```
沈黙時間率 = 沈黙時間 / 会議時間 × 100

スコア = max(0, 60 - 沈黙時間率 × 6)
```

**例**:
- 沈黙5%（3分/60分）→ スコア = 30点
- 沈黙10%（6分/60分）→ スコア = 0点

#### 4.2 沈黙の文脈（0-40点）
沈黙には「良い沈黙」と「悪い沈黙」がある:

**良い沈黙（減点なし）**:
- 考える時間（「少し考えさせてください」の後）
- 資料を読む時間（「この図を見てください」の後）
- 意図的な間（ファシリテーターの問いかけの後）

**悪い沈黙（減点対象）**:
- 質問後に誰も答えない
- 気まずい雰囲気での沈黙
- 誰が話すか様子見の沈黙

**AI分析での判定**:
```javascript
const silence_context = {
  good: [
    "少し考えさせてください",
    "資料を見てください",
    "質問はありますか？"
  ],
  bad: [
    "（沈黙）",
    "誰か意見は？",
    "どうでしょうか？"
  ]
};
```

### 計算式
```
サイレントコストスコア = 基礎点(60) + 文脈評価(40)
```

### 改善アクション

#### スコア 0-40点（要改善）
```
🚨 沈黙時間が多すぎます

【問題の可能性】
- 質問が難しすぎる
- 参加者が準備不足
- 心理的安全性の問題

【即座に実施】
1. 質問を具体的にする
   ❌「どう思いますか？」
   ✅「AとB、どちらが良いですか？」
2. 沈黙が5秒続いたら、ファシリテーターが介入
3. 事前に意見を集めておく

【期待効果】
- 会議時間を20%短縮
- 参加者の満足度向上
```

#### スコア 41-70点（改善の余地あり）
```
⚠️ 沈黙時間は許容範囲ですが、改善可能

【推奨アクション】
1. 「考える時間」は明示的に設定
   「1分間、各自で考えてください」
2. 沈黙が続く場合は、質問を変える
3. 参加者の表情を見て、理解度を確認

【期待効果】
- 会議のテンポが向上
- 参加者の集中力維持
```

#### スコア 71-100点（優秀）
```
✅ 沈黙時間が適切にコントロールされています！

【さらなる改善】
1. 「考える時間」を意図的に設ける
2. 沈黙を恐れない文化（焦って話さない）
3. 沈黙の後の発言を称賛

【期待効果】
- 深い思考が生まれる
- 質の高い意見が出る
```

## 5️⃣ アジェンダ遵守度（Agenda Adherence）

### 定義
予定していた議題から脱線せず、計画通りに進行できたかを評価する指標。

### なぜ重要か
- 脱線 = 準備していない議論 = 質が低い
- 時間配分の乱れ = 重要な議題が未消化
- アジェンダ遵守 = 事前準備の成果

### 評価項目

#### 5.1 アジェンダ消化率（0-50点）
**評価基準**:
```
消化率 = 完了したアジェンダ数 / 予定アジェンダ数 × 100

スコア = 消化率 × 0.5
```

**例**:
- 3件中3件完了 → 100% → 50点
- 3件中2件完了 → 67% → 33点

#### 5.2 脱線時間の割合（0-30点）
**評価基準**:
```
脱線時間率 = 予定外の議論時間 / 会議時間 × 100

スコア = max(0, 30 - 脱線時間率 × 3)
```

**脱線の判定**:
```javascript
// アジェンダに含まれないキーワードが出た時間
const off_topic_indicators = [
  "話は変わりますが",
  "ついでに",
  "そういえば",
  "関係ないですが"
];
```

#### 5.3 時間配分の適切性（0-20点）
各アジェンダの予定時間と実際の時間を比較:

**評価基準**:
- **18-20点**: 全アジェンダが予定時間±10%以内
- **12-17点**: 一部が予定時間を超過
- **6-11点**: 大幅な時間超過あり
- **0-5点**: 時間配分が崩壊

### 計算式
```
遵守度スコア = 消化率(50) + 脱線削減(30) + 時間配分(20)
```

### 改善アクション

#### スコア 0-40点（要改善）
```
🚨 アジェンダから大きく脱線しています

【問題の可能性】
- アジェンダが現実的でない
- ファシリテーションが弱い
- 緊急の議題が発生した

【即座に実施】
1. タイムキーパーを設定
2. 脱線したら「別途時間を設けましょう」
3. アジェンダの優先順位を明確に

【期待効果】
- 会議時間を30%短縮
- 重要な議題に集中
```

#### スコア 41-70点（改善の余地あり）
```
⚠️ 概ね予定通りですが、改善可能

【推奨アクション】
1. 各アジェンダの開始時に残り時間を宣言
2. 予定時間の80%で「まとめに入ります」
3. 未消化の議題は次回に明示的に繰り越し

【期待効果】
- 全アジェンダの消化率向上
- 参加者の時間意識向上
```

#### スコア 71-100点（優秀）
```
✅ 計画通りに進行できています！

【さらなる改善】
1. アジェンダの時間配分を過去データから最適化
2. 「バッファ時間」を意図的に設ける
3. 脱線を防ぐファシリテーション技術の共有

【期待効果】
- 会議の予測可能性向上
- 参加者の満足度向上
```

## 📈 総合評価

### 総合スコアの計算
```
総合スコア = (KPI1 × 0.25) + (KPI2 × 0.25) + (KPI3 × 0.15) + 
             (KPI4 × 0.15) + (KPI5 × 0.20)
```

**重み付けの理由**:
- 事前準備度（25%）: 最も影響が大きい
- 決定密度（25%）: 会議の本質的な目的
- アジェンダ遵守度（20%）: 計画性の指標
- 発言の多様性（15%）: 質の指標
- サイレントコスト削減（15%）: 効率の指標

### 総合評価の基準

| スコア | 評価 | 状態 |
|--------|------|------|
| 90-100 | S | 理想的な会議 |
| 80-89 | A | 優秀な会議 |
| 70-79 | B | 良好な会議 |
| 60-69 | C | 改善の余地あり |
| 50-59 | D | 要改善 |
| 0-49 | E | 抜本的な見直しが必要 |
